# AIStudioBuildWS - 产品文档

## 项目存在的意义

CLIProxyAPI 从 v6.3.x 版本开始支持通过 WebSocket 方式接入 AI Provider（首个支持 AIStudio）。然而，这种方式需要一个始终开启的浏览器来运行 AIStudioBuild 上的 WebSocket 通信程序，存在以下问题：

1. **本地部署不便**：需要保持浏览器窗口始终打开
2. **VPS 内存要求高**：传统 VPS 难以满足浏览器运行的内存需求

本项目通过使用 Docker 在 HuggingFace 上部署无头浏览器，充分利用 HuggingFace 免费实例的大内存优势，实现**零成本部署**。

## 解决的问题

1. **浏览器自动化**：自动登录并维护 Google AI Studio 会话
2. **Cookie 管理**：支持多账户 Cookie 的加载、验证和刷新
3. **WebSocket 保活**：通过定期点击页面保持连接活跃
4. **错误恢复**：自动检测登录失效并处理

## 用户体验目标

### 部署体验
- **HuggingFace 部署**：复制 Space，填写配置，一键部署
- **Docker 部署**：克隆仓库，配置 `.env`，`docker compose up -d`

### 运行体验
- **无人值守**：部署后自动运行，无需人工干预
- **多账户并行**：支持同时运行多个账户的浏览器实例
- **日志透明**：提供详细的运行日志和错误截图

### 故障处理
- **Cookie 失效检测**：自动检测并报告 Cookie 过期
- **连接异常处理**：网络错误时提供详细的诊断信息
- **优雅关闭**：支持信号处理，优雅关闭所有进程

## 核心功能模块
